## 1. 核心定义

* 浅拷贝 (Shallow Copy)：只拷贝对象本身，而不拷贝对象所指向的资源。
* 现象：拷贝后的新旧对象共享同一块内存（如动态分配的堆空间）。修改其中一个对象的内部资源，另一个也会受影响。
* 深拷贝 (Deep Copy)：不仅拷贝对象本身，还递归地拷贝对象所指向的所有资源。
* 现象：拷贝后的新旧对象在内存中完全独立。修改其中一个，另一个不会发生任何变化。

## 2. C++ 中的深浅拷贝（涉及内存安全）
C++ 默认提供的拷贝构造函数是浅拷贝（按位拷贝）。
## 浅拷贝的危机：

```
class MyBuffer {public:
    int* data;
    MyBuffer() { data = new int[10]; }
    // 默认拷贝构造函数是浅拷贝：this->data = other.data;
    ~MyBuffer() { delete[] data; } // 析构时释放内存
};
// 场景：
MyBuffer b1;
MyBuffer b2 = b1; // 浅拷贝，b1.data 和 b2.data 指向同一内存// 当函数结束时，b1 和 b2 都会调用析构函数，导致【同一块内存被释放两次】 -> 程序崩溃（Double Free）
```

## 深拷贝的实现：
必须手动编写拷贝构造函数，重新申请内存并复制内容。

```
MyBuffer(const MyBuffer& other) {
    data = new int[10]; // 重新申请空间
    memcpy(data, other.data, 10 * sizeof(int)); // 复制内容
}
```

## 3. Python 中的深浅拷贝（涉及引用计数）
Python 的赋值（a = b）本质上只是引用的传递。要实现拷贝，需要使用 copy 模块。
## 浅拷贝示例：
使用 copy.copy() 或列表切片 [:]。它只拷贝外层容器，内层嵌套的对象依然是引用。

```
import copyoriginal = [1, [2, 3], 4]shallow = copy.copy(original)

shallow[1][0] = 'X' 
print(original) # 输出 [1, ['X', 3], 4] -> 被影响了！
```

## 深拷贝示例：
使用 copy.deepcopy()。它会递归地拷贝所有嵌套对象。

```
deep = copy.deepcopy(original)
deep[1][0] = 'Y'
print(original) # 输出 [1, ['X', 3], 4] -> 不受影响
```

## 4. 面试对比总结

| 特性 | 浅拷贝 | 深拷贝 |
|---|---|---|
| 内存占用 | 较少，共享资源 | 较多，完全复制一份 |
| 安全性 | 风险高（容易出现 Double Free 或数据污染） | 安全，逻辑独立 |
| 速度 | 快 | 慢（涉及大量数据复制） |
| 适用场景 | 资源不可变或管理权明确时 | 拥有独占资源（如指针、文件句柄）时 |

## 5. 面试常考连环炮

   1. C++ 怎么避免拷贝？
      * 将拷贝构造函数声明为 = delete。
      * 使用移动语义（Move Semantics）转移所有权而非拷贝。
   2. Python 的不可变对象（如 int, tuple）拷贝时有区别吗？
      * 没有区别。Python 会为了优化性能，让深浅拷贝都指向同一个内存地址（因为它们不可变，共享是安全的）。
