原码、反码和补码是三种表示有符号整数（带正负号的数）的编码方式，本质是为了把减法变成加法，从而简化 CPU 的硬件设计。
------------------------------
# 1. 原码 (Original Code)
最直观的表示法：最高位为符号位（0正1负），其余位表示绝对值。

* 例子（假设是 8 位二进制）：
* +5 → 0000 0101
   * -5 → 1000 0101
* 缺点：
* 0 有两种表示：+0 是 00000000，-0 是 10000000。
   * 计算麻烦：不能直接把正数和负数相加（如 $5 + (-5)$ 结果不为 0），需要额外的电路处理符号位。

# 2. 反码 (Inverse Code)
为了解决原码加法的问题而提出的过渡方案。

* 规则：
* 正数：反码 = 原码（不变）。
   * 负数：符号位不变，数值位全部取反（0变1，1变0）。
* 例子：
* -5 的反码：1111 1010
* 缺点：虽然能处理加法，但 0 依然有两种表示（00000000 和 11111111）。

# 3. 补码 (Complement Code) —— 计算机真正使用的
补码完美解决了 0 的唯一性以及减法计算问题。

* 规则：
	* 正数：补码 = 原码（始终不变）。
	* 负数：反码 + 1。
* 例子：
* -5 的补码：1111 1010 (反码) + 1 = 1111 1011。
* 核心优势：
	1. 0 的表示唯一：00000000。
   2. 符号位直接参与运算：减去一个数，等同于加上这个数的补码。
   3. 多出一位表示范围：例如 8 位二进制，补码可以表示 -128 到 127（原码只能到 -127）。
* “正数-负数”的相互转换：全部取反 + 1
* 补码到原码：除符号位外取反后加1
------------------------------
# 总结口诀

* 正数：三码合一（都长得一样）。
* 负数：原码转反码（除符号位外全变），反码转补码（最后加 1）。
