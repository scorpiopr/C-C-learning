大端序（Big-Endian）与小端序（Little-Endian）  

* * *

一、 核心定義（一分鐘速記）
--------------

判斷大端和小端的關鍵，在於看多字節數據的高位字節（Most Significant Byte, MSB）存放在內存的低地址還是高地址。

以一個 32 位的十六進制整數0x12345678為例，它佔用 4 個字節。其中12是最高位字節（高位），78是最低位字節（低位）。

假設內存起始地址為0x1000：

1\. 小端序 (Little-Endian)
-----------------------

*   規則：低位字節存放在低地址，高位字節存放在高地址 
*   內存排布：
    
    地址：   0x1000    0x1001    0x1002    0x1003
    
    數據：    78        56        34        12
    
*   特點：符號直覺上是“反過來”的 。這是絕大多數個人電腦（x86/x64、現代大部分 ARM 架構）的默認處理方式

2\. 大端序 (Big-Endian)
--------------------

*   規則：高位字節存放在低地址，低位字節存放在高地址 
*   內存排布：
    
    地址：   0x1000    0x1001    0x1002    0x1003

    數據：    12        34        56        78
    
*   特點：最符合人類從左到右的閱讀習慣。這通常被稱為網絡字節序（Network Byte Order），因為互聯網 TCP/IP 協議族規定網絡傳輸一律採用大端序。

* * *

二、 大廠高频手撕代碼：如何用代碼判斷當前機器的字節序？
----------------------------

面試官常會讓你現場寫一個函數，判斷當前環境是本機大端還是小端。以下為你提供兩種標準的工業級寫法：

寫法 A：利用指針截斷（最推薦，最直觀）
--------------------

*   原理：定義一個int變量賦值為 1（0x00000001）。如果將其地址強轉為char\*（1 字節），小端機會讀到低位01，大端機會讀到高位00。
```
#include <iostream> 
bool isLittleEndian() { 
    int num = 1; 
    // 將 4 字節的 int 指針強轉為 1 字節的 char 指nil 
    char* p = reinterpret_cast<char*>(&num); 
    return (*p == 1); // 如果第一個字節是 1，說明是小端
} 
int main() { 
    if (isLittleEndian()) { 
        std::cout << "當前機器是：小端序 (Little-Endian)\n";
    } else { 
        std::cout << "當前機器是：大端序 (Big-Endian)\n";
    } 
    return 0;
}
```
寫法 B：利用聯合體（Union）（面試加分點）
------------------------

*   原理：聯合體的所有成員共享同一塊內存起始地址。
```
bool isLittleEndianUnion() {  
    union {  
        int num;  char c;
    } test; 
    test.num = 1; 
    return (test.c == 1); // 共享起始地址，直接讀取第一個字節 
}

```

三、 網絡編程中的「字節序轉換」
----------------

如果你面試的崗位涉及網絡通信，面試官一定會問你這四個標準庫函數：

*   htons()：Host to Network Short（主機字節序轉網絡字節序，16位，常用於端口號）
*   htonl()：Host to Network Long（主機字節序轉網絡字節序，32位，常用於 IP 地址）
*   ntohs()：Network to Host Short（網絡字節序轉主機字節序）
*   ntohl()：Network to Host Long（網絡字節序轉主機字節序）
---------

當本地小端機（PC）發送一個int給服務器時，必須先調用htonl()轉成網絡大端序再發送，否則服務器（大端解讀）收到數據後字節序錯亂，會解析出一個完全無關的荒謬巨數。
