## 1. 缓冲区机制
这是 stderr 与 stdout 最本质的区别：

* stdout（标准输出）：默认是行缓冲区（Line Buffered）。这意味着代码执行 printf("A"); 时，字符 "A" 不会立刻打印到屏幕上，而是**先存在内存缓冲区里。直到遇到换行符 \n、程序正常结束、或者手动调用 fflush(stdout) 时，才会统一刷新到屏幕上**。
* stderr（标准错误）：默认是无缓冲区（Unbuffered）。这意味着只要执行 fprintf(stderr, "B");，字符 "B" 会立刻、**无延迟地被刷写到输出设备（屏幕）上**，确保即使程序随后发生崩溃（Crash），错误日志也能完整呈现 [1]。

经典笔试变形题：
```
printf("A");
fprintf(stderr, "B");
```
实际输出顺序：BA。因为 "A" 被缓存在 stdout 中，而 "B" 立即从 stderr 输出，随后程序结束时才刷新输出 "A"。

## 2. 重定向（Redirection）
在 Linux 命令行运行程序时，标准输出和标准错误是可以独立分离的。

* stdout 的文件描述符（File Descriptor）是 1。
* stderr 的文件描述符是 2。

如果你不希望错误信息污染正常的数据输出，可以利用命令行将它们重定向到不同的文件：

./my_program > output.log 2> error.log

此时，所有由 printf 或 fprintf(stdout, ...) 输出的内容都会进入 output.log，而所有由 fprintf(stderr, ...) 输出的内容（比如你的 "B"）都会被单独收集进 error.log。
