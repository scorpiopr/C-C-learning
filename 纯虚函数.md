# 一、 什么是纯虚函数？
纯虚函数是一种特殊的虚函数。它在基类中只有声明，没有定义（通常情况下），并强制要求任何继承该基类的派生类（子类）必须实现这个函数，否则派生类也会变成抽象类。
## 1. 语法格式
在虚函数声明的末尾加上= 0，即表明该函数为纯虚函数。
```
  class Animal {
  public:
    // 纯虚函数
    virtual void makeSound() = 0; 
  };
```
# 二、 纯虚函数的 4 大核心特性
## 1. 导致类变成“抽象类”
包含至少一个纯虚函数的类被称为抽象类（Abstract Class）。抽象类绝对不能被实例化（不能创建它的对象）。
```
  // Animal a; // ❌ 编译错误：抽象类不能实例化对象
  Animal* ptr = nullptr; // ✅ 允许：可以定义抽象类的指针或引用，用于实现多态
```
## 2. 强制子类重写（Override）
子类继承抽象类后，必须重写所有的纯虚函数，才能实例化对象。**如果子类没有实现纯虚函数，子类也会沦为抽象类，依然无法实例化**。
```
  class Dog : public Animal {
  public:
    void makeSound() override { // ✅ 实现了基类的纯虚函数
      std::cout << "Woof!" << std::endl;
    }
  };
  Dog d; // ✅ 此时子类可以实例化
```
## 3. 纯虚函数可以有定义吗？
可以。 这是面试中非常经典的“挖坑题”。
C++ 允许为纯虚函数提供函数体（定义），但**必须在类外部定义**。即便提供了定义，该类依然是抽象类，无法实例化。它的作用是为子类提供一份“默认的兜底实现”，但子类在继承时依然必须显式重写它，并在重写中通过Base::func()来调用这份默认实现。
## 4. 虚析构函数可以是纯虚的吗？
可以，这被称为纯虚析构函数（Pure Virtual Destructor）。
如果一个类没有任何合适的成员函数可以写成纯虚函数，但你又想让它变成抽象类，就可以把它的析构函数声明为纯虚的。
唯一例外要求：如果声明了纯虚析构函数，必须在类外为它提供定义。因为子类析构时，编译器会自动由内向外调用基类的析构函数，如果没有定义，会导致链接错误。
# 三、 普通函数 vs 虚函数 vs 纯虚函数
* 普通函数（Non-virtual）：强制继承接口，且强制继承实现。子类不应该去修改它（代表不变的共性）。
* 普通虚函数（Impure Virtual）：强制继承接口，但允许重写实现。基类提供一份默认实现，子类可以不用管，也可以选择覆盖。
* 纯虚函数（Pure Virtual）：只继承接口，不继承实现。基类不负责具体怎么做，完全交给子类去个性化定制。
# 四、 纯虚函数在底层是如何工作的？
纯虚函数的底层逻辑依然依赖虚函数表（vtable）：
当一个类包含纯虚函数时，编译器为其生成的虚函数表（vtable）中，该纯虚函数对应的条目（Slot）通常会被填充为一个特殊的系统函数指针（例如 GCC 中是__cxa_pure_virtual）。
如果在运行期间，由于程序逻辑漏洞意外触发了对该纯虚函数的直接调用，程序不会正常跳转，而是会触发一个严重的运行时错误，打印类似 "pure virtual method called" 的提示并直接崩溃退出。
