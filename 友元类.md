在 C++ 中，友元类（Friend Class） 是一种允许一个类访问另一个类的私有（private）和保护（protected）成员的机制。
在面向对象编程中，类的私有成员通常是隐藏的，外部无法直接访问。但如果将类 A 声明为类 B 的“友元”，那么类 A 就可以毫无阻碍地读写类 B 中的所有私有数据。
------------------------------
## 💻 代码示例（C++17 标准）
```
#include <iostream>
#include <string>
// 声明被访问的类
class Storage {
private:
    std::string secret_data = "Top Secret Information";

    // 关键：在这里把 Accessor 声明为自己的“友元类”    
    friend class Accessor; 
};
// 声明有访问特权的友元类
class Accessor {
public:
    void printSecret(const Storage& storage) {
        // 它可以直接访问 storage 的私有变量 secret_data
        std::cout << "Accessor 成功读取: " << storage.secret_data << std::endl;
    }
};
int main() {
    Storage my_storage;
    Accessor my_accessor;
    
    my_accessor.printSecret(my_storage);
    return 0;
}
```
------------------------------
## ⚠️ 友元类的三大核心原则（面试高频考点）
在 C++ 面试中，面试官非常喜欢考察友元类的底层逻辑，你需要牢记这三点：

   1. 友元关系是单向的
   * 如果类 A 是类 B 的友元，类 A 可以访问类 B 的私有成员。
      * 但不代表类 B 自动变成类 A 的友元，类 B 不能访问类 A 的私有成员。
   2. 友元关系不能传递
   * 如果 A 是 B 的友元，B 是 C 的友元，不代表 A 是 C 的友元。
   3. 友元关系不能继承
   * 基类的友元类，不会自动变成派生类（子类）的友元类。
      * 同理，如果基类是某个类的友元，它的派生类也不会自动继承这种特权。
   
------------------------------
## 🛠️ 什么时候该用友元类？
虽然友元类破坏了封装性，但在以下场景中它是不可或缺的：

* 紧密耦合的工具类：比如一个链表类 (LinkedList) 和一个迭代器类 (Iterator)，迭代器需要频繁操作链表的私有节点指针，将其设为友元类可以极大地简化代码并提高效率。
* 模拟包/模块权限：C++ 没有 Java 的 package 访问权限，有时为了让某几个相关的类内部协同工作，会用到友元。
