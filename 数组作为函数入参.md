以二维数组 int arr[3][4] 为例
------------------------------
## 场景 1：退化为指针
这是最传统的 C 风格写法。在这两种情况下，二维数组在传入时会自动退化为指向一维数组的指针（类型为 int(*)[4]）。

* 3种函数声明：
```
void print(int arr[3][4]); // 显式写出第一维（3会被编译器自动忽略）
void print(int arr[][4]);  // 省略第一维（最常用的C风格声明）
void print(int (*arr)[4]); // 🔑 本质：普通的数组指针
```
* 2种调用该函数时的数组表示方式：
直接传入数组名，或者显式传入首行地址：
```
int my_arr[3][4] = { ... };

print(my_arr);       // ✅ 方式 1：直接写数组名（最常用，自动退化为指针）
print(&my_arr[0]);   // ✅ 方式 2：显式取第一行的地址，类型完全对等
```

------------------------------
## 场景 2：多维数组引用
如果你想在函数内部彻底***防止数组退化为指针***，并且想在函数内部利用 ***sizeof(arr) 完美算出整个数组的物理大小***，你应该使用数组引用。

* 函数声明：
```
void print_safe(int (&arr)[3][4]); // 🔑 强行锁死两维的长度
```
* 调用该函数时的数组表示方式：
只能直接传入数组名，且数组的维度必须严格匹配：
```
int my_arr[3][4];
int wrong_arr[5][4];

print_safe(my_arr);    // ✅ 方式 3：直接写数组名（安全绑定，绝不退化）//
print_safe(wrong_arr); // ❌ 编译报错！第一维长度不匹配，强类型保护
```

------------------------------
## 三、 场景 3：动态多维数组
如果你的二维数组是用 new 或者 malloc 动态开辟的“指针数组”（即数组的每一行在内存中是不连续的散落堆块），形参必须是二级指针 int**。

* 函数声明：
```
void print_dynamic(int** arr, int row, int col);
```
* 调用该函数时的数组表示方式：
直接传入二级指针名：
```
// 动态构建一个 3x4 的“伪二维数组”
int** dynamic_arr = new int*[3];
for(int i = 0; i < 3; ++i) 
  dynamic_arr[i] = new int[4];

print_dynamic(dynamic_arr, 3, 4); // ✅ 方式 4：直接传入二级指针名
```

------------------------------
## 四、 场景 4：形参为「一维普通指针」（强行扁平化摊平）
内置的多维数组在内存中其实是一行接一行、连续存放的扁平一维空间（Row-major order）。如果你想写一个通用函数，处理任意行、任意列的数组，可以把多维数组强行强转为一维指针 int* 传入。

* 函数声明：
```
void print_flat(int* arr, int row, int col) {
    // 在函数内部通过手动计算偏移量来访问元素：
    arr[i * col + j]
}
```
* 调用该函数时的数组表示方式：
必须进行强制类型转换，或者传入首元素的绝对地址：
```
int my_arr[3][4];

print_flat((int*)my_arr, 3, 4);       // ✅ 方式 5.1：强转首行地址为一维指针
print_flat(&my_arr[0][0], 3, 4);      // ✅ 方式 5.2：精准传入第 0 行第 0 个元素的地址
```

## 💡 现代 C++ 架构师的防御性建议
在真正的工业级现代 C++ 开发中，我们极其反对直接使用内置的 C 风格多维数组作为函数入参，因为它的指针退化极易引发越界雪崩。
现代标准推荐的表示方式是全面拥抱容器：

* 固定大小：使用 std::array<std::array<int, 4>, 3>。
* 动态大小：使用 std::vector<std::vector<int>>。
它们在作为参数时，可以直接像普通对象一样通过引用传参（如 const auto& matrix）进行调用，既优雅又绝对安全。
