## 一、 核心区别对比全景表

| 对比维度 | C 语言中的 struct | C++ 语言中的 struct |
|---|---|---|
| 本质定位 | 纯粹的数据封装（POD，Plain Old Data）。 | 本质就是类（Class），是面向对象的实体。 |
| 内部能否写函数 | 绝对不能（只能通过成员函数指针来曲折模拟）。 | 完全可以（支持普通函数、虚函数、构造/析构函数）。 |
| 默认访问权限 | 没有任何权限概念（隐式全公开）。 | 明确为 public（而 class 默认是 private）。 |
| 默认继承权限 | 不支持继承。 | 明确为 public 继承（而 class 默认是 private 继承）。 |
| 类型声明与定义 | 使用时必须带 struct 关键字（除非用 typedef）。 | 结构体名直接作为类名/类型名使用。 |
| 空结构体大小 | 未定义行为/由编译器决定（如 GCC 允许为 0 字节）。 | 严格为 1 字节（用于在内存中唯一标识该对象实例）。 |

## 二、 核心差异代码对比演练
## 1. 类型系统与声明的区别

* 在 C 语言中：结构体的名字并不是一个独立的类型名。如果你声明了 struct Node {...};，在定义变量时必须老老实实写完整的 struct Node my_node;。因此 C 语言中诞生了大量密密麻麻的 typedef struct {...} Node; 写法。
* 在 C++ 语言中：struct Node {...}; 声明完成后，Node 瞬间成为一个标准的类型名，你可以像使用 int 一样直接写 Node my_node;。

## 2. 空结构体（Empty Struct）的内存谜题
这是一个极高频的底层挖坑题：“一个没有任何成员变量的 struct，在 C 和 C++ 里的大小分别是多少？”

struct Empty {};


* C 语言环境：在纯 C 标准中，空结构体属于未定义行为（Undefined Behavior），但 GCC 编译器将其扩展为 0 字节。
* C++ 语言环境：sizeof(Empty) 严格等于 1 字节。因为 C++ 必须确保每个独立的类实例在内存中都拥有独一无二的内存地址（否则如果你声明一个 Empty arr[10]，所有元素的地址全都重叠了）。

## 三、 C++ 中 struct 与 class 的唯一区别
访问和继承默认权限不同
```
struct S { int a; }; // 🔑 默认是 public，外部可直接访问class C  { int a; }; // 🔑 默认是 private，外部无法直接访问
struct DerivedS : S {}; // 🔑 默认是 public 继承class DerivedC  : C {}; // 🔑 默认是 private 继承
```
除了这两个默认权限的微小差异外，C++ 的 struct 可以写显式构造函数、可以写析构函数、可以继承、可以使用 virtual 关键字写虚函数、甚至能参与运行期的多态机制，它和 class 在底层编译出来的机器码没有任何区别。
